# Security Practices

| Area | Practice | Where |
|------|----------|-------|
| Secrets | No secrets in git. DB password generated by Terraform → **Secrets Manager**; injected into ECS task defs via `secrets`/`valueFrom`. EKS uses a Kubernetes Secret (prod: sync from Secrets Manager via External Secrets). | `terraform/ecs/secrets.tf`, `kubernetes/01-config-and-secret.yaml` |
| CI auth | **GitHub OIDC → IAM role** (short-lived tokens). No long-lived AWS access keys stored in the repo. | `.github/workflows/*.yml` |
| Least-privilege IAM | ECS execution role can read **only** the one DB secret; task role is empty (app needs no AWS APIs). EKS ALB-controller role scoped via IRSA to its ServiceAccount. | `terraform/ecs/iam.tf`, `terraform/eks/alb_controller.tf` |
| Network | All tasks/pods + data in **private subnets**, no public IPs. Only ALBs are public. | `terraform/network`, `terraform/ecs/services.tf` |
| Security groups | Least privilege: each tier accepts traffic only from the tier in front (`ALB→FE→BE→data`). | `terraform/ecs/security_groups.tf` |
| Containers | **Non-root** (backend uid 1000, frontend nginx uid 101); multi-stage minimal images; `HEALTHCHECK`; k8s `allowPrivilegeEscalation:false`, dropped capabilities, read-only root FS on the backend. | `*/Dockerfile`, `kubernetes/03-backend.yaml` |
| Images | ECR **scan-on-push**, **immutable tags** (git SHA), lifecycle expiry. | `terraform/ecr/main.tf` |
| Transport | App helmet headers; ALB on :80 for the lab (add ACM + HTTPS listener for production). | `backend/src/app.js` |
| State | Terraform state in a **private, encrypted (KMS), versioned** S3 bucket with public access blocked; S3-native locking. | `terraform/bootstrap/main.tf` |

## Known lab simplifications (call these out in a real review)
- ALB serves HTTP only — production needs ACM + an HTTPS (:443) listener and HTTP→HTTPS redirect.
- The EKS `Secret` ships a demo base64 value for `kubectl apply -k`; wire **External Secrets
  Operator** (or CSI Secrets Store) to Secrets Manager for production.
- Postgres/Redis run as in-cluster containers with ephemeral storage; production → RDS + ElastiCache.
